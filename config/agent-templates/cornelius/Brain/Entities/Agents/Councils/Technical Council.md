# Technical Council

> A standing council of [[Entities/Organizations/Logix]] virtual staff. Convene it for architecture reviews and engineering decisions across the portfolio. Advisory only.

## Purpose

Review major technical decisions — architecture, data model, API standards, platform migrations, mobile/web tradeoffs. Used for architecture reviews, not strategy.

## How to convene

> "Architecture review: how should Experts App handle multi-currency FX?"

The Architect frames the design, leads weigh in from their layer, Security and DevOps stress the failure modes, and the **CTO breaks ties**.

## Decision Rules

- Majority recommendation; **CTO breaks ties**.
- Prefer simple / maintainable / proven over clever / complex / experimental.
- Challenge unnecessary complexity, duplicated logic, tight coupling.
