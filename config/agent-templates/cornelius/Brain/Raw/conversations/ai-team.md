How to create a dispatcher ?

A dispatcher is basically the “manager brain” that decides **who should respond, in what order, and with what context**.

Think of it like this:

> User → Dispatcher → Selects AI employees → Runs a structured meeting → Produces final answer

It’s not a model by itself. It’s a **routing + orchestration layer**.

### 2. Select agents

* Or a full council?

### 3. Merge responses

* Combine opinions
* Resolve conflicts
* Produce final decision

# 4. Dispatcher Logic (TypeScript Example)

This is the simplest real implementation:

```ts
type Agent = {
  id: string;
  name: string;
  role: string;
  systemPrompt: string;
};

const agents: Agent[] = loadAgents();

function classifyQuery(input: string): string[] {
  const lower = input.toLowerCase();

  const selected: string[] = [];

  if (lower.includes("architecture") || lower.includes("system")) {
    selected.push("cto");
  }

  if (lower.includes("price") || lower.includes("cost") || lower.includes("revenue")) {
    selected.push("cfo");
  }

  if (lower.includes("user") || lower.includes("ux") || lower.includes("feature")) {
    selected.push("cpo");
  }

  if (lower.includes("security") || lower.includes("auth")) {
    selected.push("security");
  }

  // fallback
  if (selected.length === 0) {
    selected.push("cto", "cpo");
  }

  return selected;
}
```

# 5. Dispatcher → Agent Execution

```ts
async function runDispatcher(userInput: string) {
  const selectedAgentIds = classifyQuery(userInput);

  const selectedAgents = agents.filter(a =>
    selectedAgentIds.includes(a.id)
  );

  const responses = await Promise.all(
    selectedAgents.map(agent =>
      callLLM({
        system: agent.systemPrompt,
        user: userInput
      })
    )
  );

  return aggregate(responses);
}
```

# 1. Agent Runtime Protocol (ARP)

This is the most important piece.

It defines **how agents talk, think, and hand off work**.

---

## Core Principle

No free-form chaos between agents.

Everything must be:

> structured → traceable → replayable

---

## Standard Message Format

Every agent message should follow this:

```ts id="arp1"
type AgentMessage = {
  id: string;
  from: string;
  to: string | "broadcast";

  type: "opinion" | "decision" | "question" | "critique" | "proposal";

  topic: string;

  content: string;

  confidence: number; // 0-1

  priority: "low" | "medium" | "high" | "critical";

  references?: string[]; // memory IDs or tool outputs

  timestamp: number;
};
```

## Example

CTO responding:


This gives you:

* auditability

* structured debates instead of random chat

Skills sit between:

Hooks make your system reactive.
