## Security Review Incident #11

### Report

Carried forward from incident #7 production CSP audit (2026-05-14). Bundle of small CSP/header hardening items that don't individually warrant their own incident but collectively close visibility and supply-chain gaps in the policy now serving from Traefik.

Low-Medium: CSP hardening follow-ups — reporting endpoint, supply-chain trims, header completeness check

#### Sub-item A — No CSP violation reporting

- **Observation:** the production CSP has no `report-uri` or `report-to` directive. CSP violations from real users (or from attackers probing the policy) are silently dropped. We will never know if a real user trips it, or if drift in a future Traefik change leaves the policy too tight.
- **Fix:**
  1. Add a tiny Next.js API route `apps/experts-app/app/api/v1/internal/csp-report/route.ts` that accepts `POST application/csp-report` and `POST application/reports+json`, validates shape minimally, and forwards to the observability pipeline (`observe("csp.violation", payload, {level: "warn"})`).
  2. Apply rate limiting (reuse `enforceUploadRateLimit`-style sliding window) keyed by IP. CSP report storms from a single client are common and would otherwise flood logs.
  3. Coordinate with platform-ops to add `report-uri /api/v1/internal/csp-report; report-to csp-endpoint;` to the Traefik `secure-headers@file` middleware, and a `Report-To: {"group":"csp-endpoint","max_age":10886400,"endpoints":[{"url":"https://experts.com.sa/api/v1/internal/csp-report"}]}` header alongside.
- **Severity:** Low individually (no exploitable consequence). High value as an observability foundation for incident #10's nonce migration.

#### Sub-item B — `font-src https://raw.githubusercontent.com`

- **Observation:** production `font-src` includes `https://raw.githubusercontent.com`. This implies a font is being fetched from a raw GitHub URL at runtime.
- **Why this is a risk:**
  1. **Supply-chain:** any change to the source repo (even by a malicious commit that survives review for a few hours) immediately affects production user-facing rendering.
  2. **GitHub TOS:** `raw.githubusercontent.com` is not a production CDN. GitHub rate-limits, sometimes serves 503, and explicitly discourages production traffic. Outages here become outages here.
  3. **Integrity:** no SRI hash, no version pin in CSP. Whatever bytes are served at fetch time get loaded.
- **Fix:**
  1. Find the font reference (likely a `@font-face` rule or a `<link rel="preload">` in a `head` injection). Grep: `rg 'raw.githubusercontent' apps/experts-app`.
  2. Pin: either self-host the font under `public/fonts/` or upload to `cdn.experts.com.sa`.
  3. Remove `raw.githubusercontent.com` from `font-src` in Traefik.

#### Sub-item C — Suspicious `script-src` and `connect-src` entries to audit

These were added at some point in CSP history and may be stale. Each needs a one-line answer from the team: "yes, we use it for X" or "drop it."

| Origin                                                                                                 | Directive   | Suspected purpose                                              | Action                                                                      |
| ------------------------------------------------------------------------------------------------------ | ----------- | -------------------------------------------------------------- | --------------------------------------------------------------------------- |
| `https://elu.dev`                                                                                      | script-src  | elu MCP / product intelligence SDK                             | Confirm SDK is still in use; if not, remove                                 |
| `https://cloudflare-dns.com`                                                                           | connect-src | DoH lookups from a third-party SDK                             | Identify the SDK; usually safe to remove if no app code uses DoH            |
| `https://www.google.com`                                                                               | connect-src | Likely reCAPTCHA                                               | If reCAPTCHA, narrow to `https://www.google.com/recaptcha/`                 |
| `https://featureassets.org` and `assetsconfigcdn.org`, `prodregistryv2.org`, `beyondwickedmapping.org` | connect-src | Statsig multi-CDN feature-gate edges (legitimate, oddly named) | Document in `docs/security/cdn-hardening.md` so future devs don't flag them |

- **Severity:** Low. Trimming reduces the attack surface (each origin in CSP is an origin an attacker can target via the same route the app trusts).

#### Sub-item D — Confirm `Strict-Transport-Security` and `Permissions-Policy` are also set

- **Observation:** these are sibling defenses to CSP, not part of CSP. Whether Traefik's `secure-headers@file` sets them is unknown from the CSP dump alone.
- **Fix:**
  1. `curl -sI "https://experts.com.sa/en/dashboard" | grep -iE 'strict-transport-security|permissions-policy'`.
  2. If `Strict-Transport-Security` is missing: add `max-age=31536000; includeSubDomains; preload` via Traefik. Consider HSTS preload list submission once stable.
  3. If `Permissions-Policy` is missing: add a baseline disabling unused powerful features — at minimum `camera=(), microphone=(), geolocation=(), payment=(self), interest-cohort=()`. Adjust for actual app needs.

#### Sub-item E — Small additions to CSP

- `upgrade-insecure-requests` — auto-upgrades any HTTP URL the app or its scripts try to load. Cheap, near-zero risk.
- Explicit `manifest-src 'self'` and `worker-src 'self'` — currently inherit from `default-src 'self'`, but explicit prevents silent regression if `default-src` is loosened later.
