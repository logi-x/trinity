# P2007 Protection - Round 2 Fixes

## Issues Found in Logs

After the initial protection, several endpoints were still experiencing P2007 errors:

1. **Prisma Import Error** - Error handler couldn't import `@prisma/client`
2. **Preview Endpoint** - `/api/v1/courses/123/preview` returning 500
3. **Modules Endpoint** - `/api/v1/courses/123/modules` returning 500
4. **Main Course Endpoints** - Still getting 500 instead of 400 for invalid UUIDs

## Root Causes

### 1. Prisma Client Import Issue

**Problem**: The error handler tried to import Prisma client types:

```typescript
import { Prisma } from "@prisma/client";
```

This caused runtime errors because Prisma client wasn't always available.

**Solution**: Changed to type-only definition:

```typescript
type PrismaClientKnownRequestError = {
  code: string;
  message: string;
  meta?: Record<string, unknown>;
  clientVersion: string;
};
```

**Files Changed**:

- `src/lib/common/errors/prisma-error.handler.ts`

### 2. Unprotected Preview Endpoint

**Problem**: No UUID validation before Prisma query:

```typescript
// Before
export async function GET(request, { params }) {
  const { id: courseId } = await params;
  try {
    const course = await prisma.course.findUnique({
      where: { id: courseId }, // ❌ Invalid UUID causes P2007
    });
  } catch (error) {
    return NextResponse.json({ error: "..." }, { status: 500 }); // ❌ Generic error
  }
}
```

**Solution**: Used route wrapper:

```typescript
// After
export const GET = withApiRoute(async (request, { validatedId: courseId }) => {
  const course = await prisma.course.findUnique({
    where: { id: courseId }, // ✅ Already validated
  });
}, "Course Preview");
```

**Files Changed**:

- `app/api/v1/courses/[id]/preview/route.ts`

### 3. Unprotected Modules Endpoint

**Problem**: Complex endpoint with no UUID validation on GET and POST.

**Solution**: Added manual validation to both methods:

```typescript
export async function GET(
  _: NextRequest,
  { params }: { params: Promise<{ id: string }> },
) {
  try {
    const { id: courseId } = await params;

    // Validate UUID format
    const paramsValidation = routeParamsWithIdSchema.safeParse({
      id: courseId,
    });
    if (!paramsValidation.success) {
      return NextResponse.json(
        {
          error: "Invalid course ID format",
          details: paramsValidation.error.format(),
        },
        { status: 400 },
      );
    }

    // Rest of the logic...
  } catch (error) {
    return handlePrismaError(error, "Course Modules List");
  }
}
```

**Files Changed**:

- `app/api/v1/courses/[id]/modules/route.ts`

## How to Protect Remaining Endpoints

### Quick Method (Use Route Wrapper)

```typescript
// 1. Import the wrapper
import { withApiRoute } from "@/lib/common/api/route-wrapper";

// 2. Replace `export async function GET` with:
export const GET = withApiRoute(async (request, { validatedId }) => {
  // Use validatedId instead of parsing params
  // ... your logic here
}, "Endpoint Name");
```

### Manual Method (Complex Endpoints)

```typescript
// 1. Add imports
import { routeParamsWithIdSchema } from "@/lib/common/validation/uuid.validation";
import { handlePrismaError } from "@/lib/common/errors/prisma-error.handler";

// 2. Add validation
export async function GET(request, { params }) {
  try {
    const { id } = await params;

    // Validate
    const validation = routeParamsWithIdSchema.safeParse({ id });
    if (!validation.success) {
      return NextResponse.json(
        { error: "Invalid ID format", details: validation.error.format() },
        { status: 400 },
      );
    }

    // Use validated ID
    const data = await prisma.model.findUnique({ where: { id } });

    return NextResponse.json(data);
  } catch (error) {
    // Handle errors
    return handlePrismaError(error, "Endpoint Name");
  }
}
```

## Lessons Learned

1. **Import types carefully** - Runtime Prisma imports can fail; use type-only imports when possible
2. **Test with real URLs** - Users will access weird URLs (like `/courses/123` or `/courses/opengraph-image1`)
3. **Validate early** - Catch invalid inputs before they reach the database
4. **Context in errors** - Makes debugging much easier
5. **Gradual rollout** - Protect high-traffic endpoints first, then expand
