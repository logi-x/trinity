## Layout direction

Use each spread as a two-page bilingual pair:

- Left page: English
- Right page: Arabic

Do not place both languages inside the same page column system. Treat each spread as mirrored twins:

- English page uses LTR composition
- Arabic page uses RTL composition
- Keep the same visual hierarchy, image system, color treatment, and content weight on both sides
- If one language runs longer, edit copy slightly instead of breaking alignment between the two pages

## Typography

- English display: DM Sans Bold
- English body: Inter
- Arabic display: Readex Pro
- Arabic body: Almarai

## Visual system

- Primary accent: `oklch(0.58 0.17 255.97)`
- Light background: `oklch(0.98 0 0)`
- Dark background: `oklch(0.08 0 0)`
- Card radius: 20px
- Media radius: 24px
- Alternate dark and light spreads
- Use product screenshots, cropped UI, and restrained abstract gradients

## Production notes for the left-right concept

- Make every spread a mirrored bilingual pair, not an asymmetrical translation page
- Keep images centered across the gutter only when they are non-textual
- Avoid paragraphs crossing the gutter
- Use separate paragraph styles for English and Arabic, even when the hierarchy matches
- For Arabic spreads, mirror alignment, icon flow, chips, and stat order
- If a spread becomes dense, reduce copy on both sides equally to preserve balance
