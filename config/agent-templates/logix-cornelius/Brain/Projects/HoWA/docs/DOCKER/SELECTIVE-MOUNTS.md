# Selective Volume Mounts for Vite HMR

## The Problem with Full Directory Mounts

### ❌ Mounting Entire App Directory

```yaml
volumes:
  - ../../apps/admin:/app/apps/admin # Overwrites EVERYTHING
  - /app/apps/admin/node_modules # Try to exclude, but doesn't work
```

**What happens:**

1. Host's `apps/admin` mounts to `/app/apps/admin`
2. This **replaces** the container's entire `apps/admin` directory
3. The anonymous volume exclusion creates an **empty** `node_modules`
4. pnpm symlinks break → `Cannot find module 'vite'`

### ✅ Selective File/Directory Mounts (The Solution!)

```yaml
volumes:
  # Only mount files that change during development
  - ../../apps/admin/resources:/app/apps/admin/resources
  - ../../apps/admin/vite.config.js:/app/apps/admin/vite.config.js
  - ../../apps/admin/tailwind.config.js:/app/apps/admin/tailwind.config.js
  - ../../apps/admin/postcss.config.js:/app/apps/admin/postcss.config.js
  - ../../apps/shared:/app/apps/shared
  # No node_modules exclusions needed - we're not overwriting the app directory!
```

**What happens:**

1. Only specific files/dirs are mounted
2. The app directory structure stays intact
3. `node_modules` remains untouched with correct pnpm symlinks
4. Everything works! ✅

## Why This Works

### pnpm Workspace Structure

```
/app/
├── node_modules/
│   ├── .pnpm/                    ← Actual packages stored here
│   │   ├── vite@7.1.12/
│   │   ├── react@19.2.0/
│   │   └── ...
│   └── .bin/
│       └── vite                  ← Binary symlink
├── apps/
│   └── admin/
│       ├── resources/            ← MOUNTED from host (HMR)
│       ├── vite.config.js        ← MOUNTED from host
│       └── node_modules/         ← FROM IMAGE (pnpm symlinks)
│           ├── .bin/
│           │   └── vite → ../../../node_modules/.bin/vite
│           ├── vite → ../../../node_modules/.pnpm/vite@7.1.12/.../vite
│           └── react → ../../../node_modules/.pnpm/react@19.2.0/.../react
```

**Key Points:**

- Root `node_modules/.pnpm/` contains actual packages
- App's `node_modules/` contains symlinks to root
- We only mount files that change (`resources/`, config files)
- We **DON'T** mount the entire `apps/admin/` directory
- The image's `node_modules` structure stays intact

## What to Mount for HMR

### For Frontend Development

```yaml
volumes:
  # Source code (changes frequently)
  - ../../apps/${APP}/resources:/app/apps/${APP}/resources

  # Build configuration (changes occasionally)
  - ../../apps/${APP}/vite.config.js:/app/apps/${APP}/vite.config.js
  - ../../apps/${APP}/tailwind.config.js:/app/apps/${APP}/tailwind.config.js
  - ../../apps/${APP}/postcss.config.js:/app/apps/${APP}/postcss.config.js

  # Shared code (used by all apps)
  - ../../apps/shared:/app/apps/shared
```

### What NOT to Mount

❌ **Don't mount these:**

- `apps/${APP}/` - Entire app directory (overwrites node_modules)
- `apps/${APP}/node_modules` - pnpm symlinks (breaks workspace)
- `node_modules/` - Root node_modules (breaks workspace)
- `apps/${APP}/package.json` - Installed in image
- `apps/${APP}/public/build/` - Generated by Vite

## Complete Example

### docker-compose.yml (Vite Services)

```yaml
services:
  # ==========================================
  # Client Vite HMR
  # ==========================================
  howa-dev-app-vite:
    build:
      context: ../../
      dockerfile: ./docker/vite/Dockerfile
      target: development
      args:
        APP_PATH: client
    image: loogix/howa-app-vite:dev
    container_name: howa-dev-app-vite
    working_dir: /app/apps/client
    command: ["pnpm", "dev:app"]
    volumes:
      # Selective mounts - only what changes
      - ../../apps/client/resources:/app/apps/client/resources
      - ../../apps/client/vite.config.js:/app/apps/client/vite.config.js
      - ../../apps/client/tailwind.config.js:/app/apps/client/tailwind.config.js
      - ../../apps/client/postcss.config.js:/app/apps/client/postcss.config.js
      - ../../apps/shared:/app/apps/shared
    ports:
      - "5175:5175"
    environment:
      - NODE_ENV=development
    networks:
      - howa-shared-network

  # ==========================================
  # Admin Vite HMR
  # ==========================================
  howa-dev-core-vite:
    build:
      context: ../../
      dockerfile: ./docker/vite/Dockerfile
      target: development
      args:
        APP_PATH: admin
    image: loogix/howa-core-vite:dev
    container_name: howa-dev-core-vite
    working_dir: /app/apps/admin
    command: ["pnpm", "dev:app"]
    volumes:
      # Selective mounts - only what changes
      - ../../apps/admin/resources:/app/apps/admin/resources
      - ../../apps/admin/vite.config.js:/app/apps/admin/vite.config.js
      - ../../apps/admin/tailwind.config.js:/app/apps/admin/tailwind.config.js
      - ../../apps/admin/postcss.config.js:/app/apps/admin/postcss.config.js
      - ../../apps/shared:/app/apps/shared
    ports:
      - "5176:5176"
    environment:
      - NODE_ENV=development
    networks:
      - howa-shared-network
```

## When to Rebuild

### ✅ Rebuild Required

```bash

# Changed pnpm-lock.yaml
```

### ❌ No Rebuild Needed

```bash
# Changed .tsx/.jsx files in resources/
# Changed .css files in resources/

```

## Troubleshooting

### If you add a new config file

```yaml
# Need to mount it explicitly
volumes:
  - ../../apps/admin/resources:/app/apps/admin/resources
  - ../../apps/admin/vite.config.js:/app/apps/admin/vite.config.js
  - ../../apps/admin/my-new-config.js:/app/apps/admin/my-new-config.js
  # ^ Add this line
```

### If HMR doesn't detect changes

```bash
# 1. Check if the file is mounted
docker compose exec howa-dev-app-vite ls -la /app/apps/client/resources/js/

# 2. Check Vite logs
docker compose logs -f howa-dev-app-vite

# 3. Restart container
docker compose restart howa-dev-app-vite
```

## Comparison: Anonymous Volumes vs Selective Mounts

### Approach 1: Full Mount + Anonymous Volume Exclusions (Doesn't Work)

```yaml
volumes:
  - ../../apps/admin:/app/apps/admin # Mount everything
  - /app/node_modules # Try to exclude root
  - /app/apps/admin/node_modules # Try to exclude app
```

**Result:** ❌ Anonymous volume creates **empty** node_modules, symlinks break

### Approach 2: Selective Mounts (Works Perfect!)

```yaml
volumes:
  - ../../apps/admin/resources:/app/apps/admin/resources
  - ../../apps/admin/vite.config.js:/app/apps/admin/vite.config.js
  - ../../apps/admin/tailwind.config.js:/app/apps/admin/tailwind.config.js
  - ../../apps/shared:/app/apps/shared
```

**Result:** ✅ Only specific files/dirs mounted, node_modules stays intact

## What Gets Mounted vs What Stays in Image

### Mounted from Host (Hot Reload)

- ✅ `resources/` - All frontend source code
- ✅ `vite.config.js` - Vite configuration
- ✅ `tailwind.config.js` - Tailwind configuration
- ✅ `postcss.config.js` - PostCSS configuration
- ✅ `apps/shared/` - Shared components/utils

### Kept from Image (Pre-installed)

- ✅ `node_modules/` - Root pnpm store with all packages
- ✅ `apps/${APP}/node_modules/` - App symlinks to root
- ✅ `package.json` - Dependency manifest
- ✅ `pnpm-lock.yaml` - Locked versions
- ✅ `apps/${APP}/public/` - Public directory
- ✅ `apps/${APP}/bootstrap/` - Laravel bootstrap (SSR)
