# PRD

PRD means product requirements document: the artifact that captures problem framing, scope, expected behavior, and acceptance direction before implementation.

## Role in this workspace

PRDs show up most often in planning-heavy conversations and agent workflows, especially where tasks are broken into phases and checked against explicit requirements.

## What a PRD is doing

- Defines the problem and target outcome
- Lists key requirements and constraints
- Gives implementation work a scope boundary
- Provides a basis for review and completion criteria

## Relationship to execution

A PRD is upstream of delivery tools. The PRD defines what should happen; downstream execution then governs how that work is staged and tracked.

Because this vault stores planning notes, agent prompts, and project docs, PRD references are worth keeping as a reusable concept rather than letting them disappear into conversation history.
