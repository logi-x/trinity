# TypeScript

TypeScript is the default language for active Experts App product code. Most references here are really about modeling product state clearly and making contracts safer across routes, components, and database boundaries.

## Why it matters

- catches contract drift earlier
- makes complex UI and billing flows more explicit
- improves maintainability of shared modules and route handlers
- exposes places where product state is underspecified

## Typical conversation themes

- interface shape drift
- handler return types
- component prop modeling
- schema-generated types versus handwritten domain types
