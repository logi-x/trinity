# Cursor Rules

Cursor rules are the project-specific conventions mirrored into the vault from repository `.cursor/rules` folders so they can be searched, cited, and reused outside the editor.

## Why they matter

They act as the practical operating system for agent work: naming rules, API conventions, data-fetching patterns, documentation expectations, and project structure guidance.

## Typical content

- architecture and project structure
- naming conventions
- frontend and API rules
- i18n and RTL guidance
- documentation and testing standards

## Relationship to agents

These rules complement the broader agent guidance in [[Entities/Agents]] and the reusable prompt structure in [[Agents/Prompt Template]]. They are narrower, repo-aware, and meant to reduce ambiguity during implementation.
