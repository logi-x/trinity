# EXP-204: getBaseUrl() staging URL hardcoded fallback

**Linear:** [EXP-204](https://linear.app/experts/issue/EXP-204) | **Status:** Resolved (PR #665)

## Summary
`getBaseUrl()` in `src/lib/utils.ts` fell back to `https://staging.experts.com.sa` when `NEXT_PUBLIC_APP_URL` was not set at build time. Production Docker builds that did not pass `--build-arg NEXT_PUBLIC_APP_URL=...` silently minted URLs pointing at the staging environment.

## Impact
All absolute URLs generated by the app (OG images, email links, sitemap, share links) pointed at staging when the env var was absent. Silent misconfiguration — no runtime error.

## Fix
PR #665 established `src/lib/config/app-url.ts` as the canonical resolver. `getAppBaseUrl()` throws in production if `NEXT_PUBLIC_APP_URL` absent (fail-closed). `getPublicBaseUrl()` returns `undefined` in SSG/build contexts. ESLint rule bans `staging.experts.com.sa` literal in `app/**` and `src/lib/**`.

PR #643 (attempt 1) used an `APP_BASE_URL` env var fallback — rejected because any fallback permits misconfigured builds to continue silently.
