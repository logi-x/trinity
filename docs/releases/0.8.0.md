## Notable in this release

- **Security & auth hardening** — JWT logout revocation (#187), enumeration-uniform 404 responses (#186), credential-encryption-key rotation (#267), and remediation of 23 Dependabot alerts on the default branch (#1353).
- **Enterprise identity & access** — two-factor auth (2FA/TOTP), SSO (SAML/OIDC), the Access tab for managing operator access, and a fleet-wide agent-to-agent permissions matrix (#388, enterprise#32, enterprise#17, enterprise#84).
- **Outbound voice across channels** — agents reply with ElevenLabs TTS voice notes on Telegram, Slack, and WhatsApp (enterprise#24 epic, #25/#26/#56).
- **Brain Orb** — the self-rendering Mind page for Cornelius-class agents: live voice tile, KB-write actions, fork-to-own creation, and the full capture→re-export→refetch loop (enterprise#58/#60/#61/#93 + fixes).
- **Reliability** — correlated-failure / thundering-herd controls (#1085), effect-scoped idempotency (#1084), recovery-trace injection (#1401), and fixes for circuit-breaker false-opens on busy agents (#1463), per-worker monitoring duplication (#1464), and stale `next_run_at` skipped-fire schedules (#1472).
- **Agent loops** — hard cost budget (#1155), wall-clock deadline (#1156), and no-progress/doom-loop detection (#1157).
- **UI** — unified Chat + Session tab (#1112), the "Grid" fleet dashboard view (enterprise#47), a guided first-install onboarding wizard (enterprise#52), Trinity browser-tab branding + per-route titles (#1416/#1418), and lightweight dashboard loading states (#1266).
- **Webhooks** — optional per-schedule HMAC signature auth plus pre-auth rate limit, body cap, and soft-deleted-agent guard (enterprise#77, #1423/#1424).

## Upgrade notes

- **Two-factor auth (#388)** is **mandatory for admin accounts** (optional for other users) once the enterprise module is entitled — admins are prompted to enroll TOTP on next login.
- **First-time setup changed** (enterprise#49): the setup-token field is gone; the wizard now requires an admin email and captures org + updates opt-in. Deploy the first-run window behind a tunnel/VPN until setup completes.
- Re-login after upgrade (JWT `SECRET_KEY` rotates on backend restart); reconnect MCP clients (`/mcp`).
- Carried forward from v0.7.0: PostgreSQL is the recommended backend for production (SQLite end-of-support Sep 1 2026 — `docs/migrations/SQLITE_TO_POSTGRES.md`); `AGENT_AUTH_SECRET` and Redis ACL passwords are required (`start.sh` auto-generates them).
