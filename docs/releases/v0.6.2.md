## ⚠️ Heads-up: SQLite end-of-support — September 1, 2026

**PostgreSQL is the forward path; SQLite support ends September 1, 2026.**

Trinity has supported PostgreSQL alongside SQLite since the dual-track database
work (#300, #1183). We're committing to a single supported backend:

- **SQLite reaches end-of-support on September 1, 2026.**
- SQLite remains the **zero-config default until then**, and existing installs keep
  working — but after the date SQLite stops receiving schema migrations and fixes.
- Operators running the default SQLite setup should **plan to move to PostgreSQL
  before the cutoff.**

👉 **Migration guide:** [SQLite → PostgreSQL Migration](../migrations/SQLITE_TO_POSTGRES.md)
— how to switch (`DATABASE_URL` + the bundled `postgres` profile), how to migrate an
existing deployment, and how to verify the cutover.

A prominent in-repo reminder (CLAUDE.md / README.md / AGENTS.md) tracks alongside
this announcement until the EOL date (#1279).
