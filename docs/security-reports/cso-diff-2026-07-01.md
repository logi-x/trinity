### Core security posture (verified CLEAN, with proof)
- **Auth boundaries** — every mutating/config route is `OwnedAgentByName` (`/scope`, `/action`, `/actions`, `/refresh`, `/postprocess` GET+PUT); reads are `AuthorizedAgentByName`. Agent-server routes sit behind the #1159 `X-Trinity-Agent-Token` middleware (no unauth surface).
- **can_write split airtight** — `voice-token` mint (AuthorizedAgentByName, shared users) computes `can_write = BRAIN_ORB_WRITE_ENABLED and db.can_user_share_agent(...)` — the *same* owner/admin predicate as `OwnedAgentByName`. Shared users get the read-only manifest AND a hard 403 at `/action`; the JWT lives in the parent orb page, not the voice iframe.
- **Injection** — the exec path (`claude -p`) is forked inside the agent hook, run via `_run_hook` = `create_subprocess_exec` + **stdin** (no shell string; no user/agent input on a command line). Transcript piped on stdin; args/paths never interpolated by Trinity.
- **Enum gate** — `_ALLOWED_ACTIONS = {capture, link, capture_transcript, process_transcript}`; `run_skill` (arbitrary exec) rejected 400 before any forward.
- **Idempotency** — begin → replay(snapshot)/in_flight→409 → rate-limit → forward → complete; `fail()` on every error path; key folds the verb (`brain_orb_action:{action}:{key}`).
- **Frontend** — no `v-html` in the new surface; toasts/errors use `textContent`/text interpolation; all three inbound `postMessage` handlers pin `e.origin === window.location.origin`.
