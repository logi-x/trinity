## What was checked and cleared

- **Injection (A03):** the insert is SQLAlchemy core `insert().values()` (fully
  parameterized); both migrations use constant DDL (`ALTER TABLE agent_loops ADD
  COLUMN ...`) with no user input. No SQL/command/template injection vector.
- **Access control (A01):** no auth changes. Start/list gated by
  `get_authorized_agent`; get/stop by `_check_loop_access`. The new read fields
  expose only the caller's own loop cost.
- **Input validation:** `max_cost_usd: Optional[float] = Field(default=None,
  gt=0)` rejects zero/negative; Pydantic rejects `NaN` (`NaN > 0` is False). A
  caller-supplied `inf` passes `gt=0` but is **harmless** — `accumulated_cost >=
  inf` never trips, so the loop simply falls back to the `max_runs` (1–100) hard
  stop. No unbounded-execution path.
- **Cost-accumulation integrity:** the runner counts only
  `math.isfinite(c) and c > 0`, so a `NaN`/`inf` reported cost cannot poison the
  accumulator; `None`/unknown cost is fail-open (counts as 0) and WARN-logged
  when a budget is active.
- **Supply chain (Phase 3):** no new deps; `math` (stdlib) is the only added
  import. No install scripts.
- **Secrets / crypto / external calls / file writes:** none touched.
