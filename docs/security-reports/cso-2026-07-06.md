**F1** — the user's PAT lives in the agent env and origin remote URL; a prompt-injected agent can read its own credential. A classic `repo`-scope PAT therefore escalates from "one workspace" to "whole GitHub account". Mechanism is the documented #347 per-agent PAT path (AES-256-GCM at rest); the MCP tool is deliberately excluded (args are audit-logged). **Remediation applied**: create-modal PAT help now leads with a fine-grained single-repo token recommendation and states that the agent can read its own git credential.
