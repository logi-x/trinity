## Attacker analysis (Phases 1, 5, 7, 9)

- **A01 Broken Access Control** — the governor gate sits *after* `authorize_heartbeat`
  (the agent's own agent-scoped MCP key, pinned to the path `agent_name`) and the
  `execution.agent_name == agent_name` ownership check. An unauthenticated or
  cross-agent caller is rejected (403/404/409) before any governor code runs. ✓
- **Pause weaponization** — *verified non-finding* (independent fresh-context refuter,
  CONFIRMED). The detector counts **distinct agents** (`ZADD {agent_name: now}` →
  `ZCARD`), so one compromised identity contributes at most 1 toward the threshold of
  20. `agent_name` is the server-side `apply_result` param (never an attacker body
  field), and recording is gated on the CAS `won` bool (no replay double-count).
  Arming requires 20 genuinely distinct compromised agent keys — and even then the
  effect is only to **delay** (never drop) terminals behind an auto-expiring 300s
  fail-open pause, with the feature default-OFF.
- **Terminal-drop risk** — the throttle/pause returns **503** (∉ the agent's
  `_PERMANENT_STATUSES`), so a throttled callback stays persisted on disk and retries;
  the startup sweep + lease reaper are never-drop backstops. `rate_limiter.check`
  confirmed fail-open (in-process fallback on Redis outage). No completed work is lost.
- **Header / Redis-key injection** — `Retry-After` is `str(int)`; rate-limit keys embed
  the auth-validated `agent_name` (Redis keys are binary-safe). No injection. ✓
- **Info disclosure / logging** — 503 detail strings are generic; logs carry
  `agent_name` + counts, no secrets. Feature-flag echo is a non-sensitive bool. ✓
- **`random` for jitter** — not security-sensitive (no unpredictability requirement);
  not a finding.
