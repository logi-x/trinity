## What changed

Issue #792 adds a SUB-003 subscription **switch + one-shot retry** to
`TaskExecutionService.execute_task` (`src/backend/services/task_execution_service.py`).
When the agent returns a 429 / auth-class response, the service now intercepts it
*pre-raise* (mirroring the #678 reader-race path), triggers the existing SUB-003
auto-switch, and — if the switch succeeds — re-issues the turn **once** with the
**same `execution_id`** so a one-shot trigger (manual/webhook/mcp) recovers instead of
landing `FAILED`.

New code: `_extract_agent_error()` (shared body parser), `classify_switch_failure()`
(status/​body → `failure_kind`), `_salvage_attempt_cost()` (cost rollup), the pre-raise
switch+retry block, and a `subscription_switch_attempted` cascade guard read by the
`except httpx.HTTPError` handler. The except handler was refactored to reuse the shared
extractor and to skip a second switch when the pre-raise path already attempted one.

The remaining changed files are **docs** (excluded, FP rule #12) and **one pure unit
test** with mocked deps and no non-test wiring (excluded, FP rule #8).

## Attack-surface delta

- **No** new endpoints, routes, auth gates, or user-facing input.
- Only **new untrusted input**: the agent's HTTP response body, parsed via httpx
  `response.json()` / `response.text[:500]`. The agent is a semi-trusted container
  running LLM code.
- Only **new outbound call**: one re-issued `POST /api/task` to the **same** agent,
  timeout-capped.

## Candidates considered and why each was discarded

1. **Cost / LLM amplification via the new retry (Phase 7; the one DoS exception).**
   Control flow is linear and bounded: original call → optional #678 reader-race retry
   (one-shot, owns `retry_count`) → optional #792 switch retry (one-shot, owns the
   separate `subscription_switch_attempted` flag). Worst case **3 agent calls per
   execution**, no loop, no re-entry. Each retry timeout is capped at
   `_AUTO_RETRY_MAX_TIMEOUT_S` (300s); the switch retry additionally caps to the
   *remaining* original budget (`max(1.0, effective_timeout - elapsed)`). A malicious
   agent forcing the path burns only its **own owner's** subscription/budget
   (single-tenant). **Bounded one-shot → not the unbounded-cost exception. Discarded.**

2. **Forced subscription switch via crafted response body** (agent returns
   401/402/403 + auth-tripping body to provoke a SUB-003 switch). The switch machinery
   is pre-existing; #792 only broadens *which* responses trigger it and moves one
   trigger pre-raise. The switch is scoped to the agent's owner — no cross-tenant
   confidentiality/integrity impact. Falls under bounded availability of the owner's own
   resources. **Discarded** (hard exclusion #1; no cross-tenant path).

3. **Secret leakage via agent error body into audit/notification/logs.**
   `switch_error_msg` (from the agent body) flows into
   `handle_subscription_failure(error_message=…)` → audit + notification text. This is
   **identical to pre-existing behavior** in the `except` handler (same body, same sink);
   #792 adds a second call site with the same data, not a new leakage class. The new
   `auto_retry` audit entry logs only `failure_kind`/`new_subscription`/`phase` — **not**
   the agent body. Single-tenant. **Discarded** (pre-existing; FP rule #7 spirit).

4. **Cost trust via `_salvage_attempt_cost` reading agent-supplied `cost_usd`.**
   Agent self-reported cost is already the trust model for all cost accounting; the value
   feeds the owner's own spend rollup, guarded `> 0`. **Discarded** (trusted-channel,
   no security impact).

## Positive notes

- The change **adds** an `auto_retry` audit-log entry (A09 improvement) and keeps the
  agent-controlled error string out of its `details` payload.
- Body parsing uses httpx's safe `.json()` / truncated `.text` — no `eval`,
  `pickle`, `yaml.load`, SQL, `subprocess`, or URL-from-user-input introduced.
- Same-`execution_id` re-issue does not double-finalize: the first (failed) attempt never
  reached the CAS-guarded `apply_result`, and `previous_attempt_cost` rolls the first
  attempt's spend into the terminal write.
