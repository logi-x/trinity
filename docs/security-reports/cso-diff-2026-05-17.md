**L-1 — Untracked `uv.lock` is not gitignored** (`uv.lock`)

The new 3-line `uv.lock` contains only `version`, `revision`, `requires-python` — an empty placeholder generated by `uv` against this repo's pytest-only `pyproject.toml` (no `[project]` or `[tool.uv]` section to lock against). It has no `[[package]]` entries, so it introduces zero dependency-supply-chain surface. However:

- `git check-ignore uv.lock` → not ignored
- `.gitignore` has no `uv.lock` entry
- The file has never been committed in real (non-checkpoint) history
- A careless `git add .` would stage this empty lockfile

Not a security vulnerability — flagged as repo hygiene. If you don't intend to adopt `uv` for backend deps in this PR, either delete the file or add `uv.lock` to `.gitignore`. If you do intend to adopt `uv`, the lockfile needs real `[[package]]` entries before merge so future supply-chain audits have something to verify against.

## Notes for the PR author

Positive observations (not findings, just confirming the PR does what it claims):

- All three test files set a fresh `secrets.token_hex(32)` `CREDENTIAL_ENCRYPTION_KEY` per test via `monkeypatch` autouse fixture — no shared key across tests, no hardcoded key material.
- Test fixtures are unambiguously fake: `FAKE-TEST-FIXTURE-not-a-real-token`, alphabet-walked filler (`aaaa…jjjj…`), and named patterns (`helper-roundtrip-…`, `wrong-key-test-…`). A future grep for real-format tokens would never collide.
- The "wrong key" tests rotate the env mid-test via `monkeypatch.setenv` — verifies the AES-256-GCM auth-tag rejection path that protects against key-substitution attacks.
- Twilio AccountSid plaintext invariant is explicitly pinned (`test_account_sid_stays_plaintext`) — good defensive test against a future refactor that might accidentally encrypt the public identifier (which would break Twilio API calls without any obvious symptom).
- Slack plaintext-fallback path (`slack_channels.py:47-49`) is tested for both the data path (legacy `xoxb-*` rows still readable) and the operator-visibility path (WARNING log fires).
