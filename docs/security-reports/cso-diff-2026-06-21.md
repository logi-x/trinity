#### Notes / positive observations

1. **Pure structural refactor — no behavioral change.** `AUTH_INDICATORS`, `NON_AUTH_KILL_MARKERS`, and `is_auth_failure` were lifted verbatim into `failure_classifier.py`. The #904 kill-marker short-circuit (SIGKILL/OOM/timeout → not an auth failure) is preserved exactly. Verified by SHA-256 equality of the two copies (`1c345c4a…`) and line-for-line comparison against the deleted code.

2. **Security-relevant gate intact.** `is_auth_failure` gates SUB-003 subscription-credential auto-switch. Both consumers (`routers/chat.py:925`, `services/task_execution_service.py:1155`) still `from services.subscription_auto_switch import is_auth_failure`, which now re-exports from the canonical module (`from services.failure_classifier import is_auth_failure as is_auth_failure`). Import path and patch targets unchanged → no regression in the auth-class classification that drives credential switching.

3. **Injection surface reduced (improvement).** The old test executed a text slice of `scheduler/service.py` via `exec(compile(snippet, …))` (carried `# noqa: S102`). The rewrite removes that entirely and loads both pure files via `importlib.util` — no `exec`/`compile` of source anywhere in scope. Zero injection primitives (`eval`/`exec`/`os.system`/`subprocess`) in the changed code.

4. **No suppressions added.** No new `# noqa`, `# type: ignore`, or `# nosec` introduced; the single removed one (`S102`) is a net reduction.

5. **Drift guard hardens the invariant.** `TestBackendSchedulerParity` (byte-identity) + `TestBackendReExportGuard` (no redefinition / re-export pinned) prevent the two copies from silently diverging — which is exactly how the #904 bug class would re-appear on the scheduler path. This is a security-posture *improvement* over the prior "keep in sync by hand" comment.

### Recommendation
**CLEAR** — no CRITICAL or HIGH findings. The change is a behavior-preserving consolidation of an existing classifier, verified byte-identical, with the security-relevant SUB-003 gate intact and the injection surface reduced. Safe to merge from a security standpoint.
