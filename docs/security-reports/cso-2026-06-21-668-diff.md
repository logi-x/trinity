## Findings

### [1] LLM cost amplification — `?include_ai=true` has no rate limit — MEDIUM (8/10, VERIFIED)
- **File**: `src/backend/routers/compatibility.py:37`; fan-out at `services/compatibility/ai_checks.py:222`.
- **Issue**: `include_ai=true` triggers ~8 concurrent Anthropic (Haiku) calls; the endpoint has no `enforce(...)` gate. The persisted-cache design protects the default (`include_ai=false`) path, but the forced-refresh path is unbounded per authenticated user.
- **Exploit**: an authenticated user with access to an agent loops `GET …/compatibility?include_ai=true` (or the MCP tool, which defaults `include_ai=true`); each call bills ~8 Haiku calls (~96K input tokens) on the platform's Anthropic key.
- **Impact**: bounded cost per request (Haiku is cheap; bundle capped at 48 KB), unbounded across requests by an authenticated user.
- **Fix (applied)**: gate the `include_ai` branch with the existing shared limiter — `enforce(f"compat_ai:{user}:{agent}", limit=5, window_seconds=60)` (`services/rate_limiter.py`). Returns 429 + `Retry-After`.

## Clean (verified, not assumed)
- **A03 Command injection** — `git_dir` is one of two fixed strings (`_detect_git_dir`); script/content base64-injected (shell-safe charset); `agent_name` is a Docker API arg, never shell-interpolated, DB-validated by the auth dependency.
- **A10 SSRF** — only outbound URL is the hardcoded `api.anthropic.com`; exec target is the validated `agent-{name}`.
- **AI data egress** — `.env`/generated `.mcp.json` are existence-only (content `None`, excluded from the AI bundle); `_AI_SAFE_FILES` excludes them; `_redact` strips secret patterns before egress.
- **XSS** — AI explanations rendered via `renderMarkdown` (DOMPurify) — `CompatibilityPanel.vue:204`.
- **A01 Access control** — read = AuthorizedAgentByName; fix = OwnedAgentByName; `check_id` whitelisted against `spec.AUTO_FIXABLE_IDS`; fix logic is a fixed registry, not user input. No IDOR/EoP.
- **A03 SQLi** — SQLAlchemy Core (parameterized). **A08** — `yaml.safe_load` only. **ReDoS** — no catastrophic regexes; input size-capped.
